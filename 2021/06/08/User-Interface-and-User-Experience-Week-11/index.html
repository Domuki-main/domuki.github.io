User Interface and User Experience Week 11

Author: Domuki Date: June 8, 2021  17:34:06 Category: UIUX

Test which is the last part of design thinking.

Because there is a possibility of returning to research and ideas, the process of design thinking may not end.
Since something never ends, I always work in an infinite loop until I solve the problem defined first.

Testing can be performed in many ways such as:

  • Screen ( Test screen in front of audience )
  • Launch ( Website lunched , debugging )
  • Publish ( books published )
  • Print ( Printing brochures )
  • Debug ( Debugging the software )
  • Ship ( Shipping the products )
  • Promote ( promoting the product )
  • Release ( Release it into the world )
  • Beta ( if its software we release the beta version first )

techniques:

  • Low fidelity: Create prototype using paper and then test it and it should be done with your team members.
  • Usability testing: Testing the designed user interface
  • Think-aloud walkthrough for usability testing: going through app/web user interface and while performing different we should say it loud on actions that we are currently performing.
  • Heuristic Evaluation: During this technique we have to choose a heuristic guideline such as Jakob Nielsen’s 10 general principles. then define a rating system and then choose a proper method for documentation.
