Environment variable configuration

Open Advanced system setting, click “Environment variable”

New a variable called “JAVA_HOME”

Variable name: JAVA_HOME

Variable value: the path where you installed java

Edit “Path”, then Add 2 values

%JAVA_HOME%\bin

%JAVA_HOME%\jre\bin

New a variable called “CLASSPATH”

Variable name: CLASSPATH

Variable value: .;%JAVA_HOME%\lib;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar

Testing

Enter cmd, input code: java -version

Explanation

JAVA_HOME

Variable name: JAVA_HOME
Variable value: C:\Program File\Java\jdk1.8.0_281
Purpose: define a variable for use in other places

Path

Variable name: Path
Variable value: %JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;
Purpose: Make the system recognize java, javac, javap and other commands under any path

CLASSPATH

Variable name: CLASSPATH
Variable value: .;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar
Purpose: Tell the JVM to use or execute the class on which path, so that the JVM can load the class file, .; indicates the current path, tools.jar and dt.jar are the class library paths
