The reflection of Java

Author: Domuki Date: March 19, 2021  17:14:47 Category: Basic Java

The Java reflection is in the running state, for any class, you can know all the properties and methods of this class; for any object, you can call any of its methods and properties.

The most important use of reflection is to develop various general frameworks

  • Figure out the class of any object at runtime
  • Construct an object of any class at runtime
  • Figure out the member variables and methods of any class at runtime
  • Call any object’s method at runtime

Relative Class of Reflect

Class name Function (presenting)
Class The entity of the class, representing the class and interface in the running Java application
Field Class member variables
Method Methods in class
Constructor Constructor in class

Field

Method Usage
equals(Object obj) Returns true if the attribute is equal to obj
get(Object obj) get the corresponding attribute value in obj
set(Object obj, Object value) Set the corresponding attribute value in obj

Method

Method Usage
invoke(Object obj, Object… args) Pass the object object and parameters to call the corresponding method of the object

Constructor

Method Usage
newInstance(Object… initargs) Create objects of the class based on the passed parameters
