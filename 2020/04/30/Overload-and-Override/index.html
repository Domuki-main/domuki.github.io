Overload and Override

Override and Overload which always be confounded are two different means in Java. In order to distinguish these two means better, this essay had been written.

1. Overload

Definition: Different methods to have the same name, but different signatures where the signature can differ by the number of input parameters or type of input parameters or both.

Target: Methods (parameters)

2. Override

Definition: a feature that allows a subclass or child class to provide a specific implementation of a method that is already provided by one of its super-classes or parent classes.

Target: Methods (visiting right & return type)

Tips: you can use @Override comment to help for checking.

    public > protected > (default) > private

  2. Changing return type

    If you change the return type of the method2, the new type’s level cannot higher than parent class.

    Object > String
