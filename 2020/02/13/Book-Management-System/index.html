1. Introduction

1.1. Purpose

In first term, we just studied from Chapter 1 to Chapter 8 which are quite simple part on textbook of Java programming. So it means that I cannot operate SQL data base or something like that. The only things I can use to store the data is TXT file, according to Chapter 8, String and File Operation.

For making book management system this time, I watched a lot of information and some projects from others which make me confused.

Therefore, I decided to make it by myself, because I’m too lazy, too lazy to read others’ codes.

Now this project was completed. Although, it’s not perfect enough but really successful to me.

1.2.2. The Objects

1.2.2.1. User

​ User is a object to describe the users of this system, and there’re many kinds of users.

1.2.2.2. Book

​ Book is a object to describe the books of this system, and there’re many kinds of books.

1.2.2.3. History

​ History is a object to describe an event which is used to record the actions, like lending books or returning books.

1.2.3. About User

1.2.3.1. Basic Parameters of User

​ There’re 3 parameters of User, username, password and identity.

​ username and password are used to Login, Register and Create issue

​ identity is used to distinguish different kinds of users, like Super Administer*, *Administer and Normal User

1.2.3.2. What can Users do?

Users can do many things as they want, like borrowing books and returning books. And they can watch their history operations on this program.

1.2.4. About Book

1.2.4.1. Basic Parameters of Book

There’re 5 parameters of User, book name, author, classify, amount and identity.

book name just for name of books, obviously.

author and classify are the information of book

amount is the quantity of book, because one kind of book doesn’t mean there’s only one book.

identity for searching of books, books’ name might be same but identities are unique.

1.2.4.2. The Actions for Book

​ 1. Add Book: add new book to library

​ 2. Delete Book: delete book existing in library

​ 3. Change Book: change the parameters of book

​ 4. Lend Book: quantity minus

​ 5. Return Book: quantity add

1.2.5. About History

1.2.5.1. Basic Parameters of History

There’re 5 parameters of User, userLend, bookLend, date, amount and state.

userLend is the user who operated (it just for the lender at the beginning but now the meaning are changed)

bookLend is the book which was operated (it just for the lender at the beginning but now the meaning are changed)

data is the time when the issue happened

state is the kind of operation, like LEND, RETURN, ADD etc.

amount is the quantity of the book which was operated

1.2.5.2. The Actions for History

​ 1. Add History: add new event

​ 2. Delete History: delete event existed

​ 3. Change History: change the parameters of History

1.2.6. About Importing and Storing

1.2.6.1. Importing

Input all data of TXT into RAM to create a new String named “TOTAL STRING“.

Then delete the useless string in TOTAL STRING like parameters’ name to create FIRST STRING which only have the information fitting in defined array like BOOKLIST[].

After that, split the FIRST STRING by “\n” and put all independent String in a String array like STRING[]. one row one book.

Split the STRING[] by “\t” for every index and put it into the BOOKLIST[].

1.2.6.1. Storing

Use FOR loop and PrintWriter to store the information.
