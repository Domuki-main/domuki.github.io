Java Learning Chapter 2 Problem Analyse

Author: Domuki Date: February 12, 2020  21:54:51 Category: Java Study

Learning Outcome

At the end of this chapter, student should be able to:

​ Explain the step by step in problem solving (C1, A3)

​ Analyse problem and identify the problem requirements (CTPS)

​ Using the correct symbol/notations in developing a flow chart (P3)

​ Write a simple program to display the output (A2)

What Problem Can Be Solved By Computer

• When the solution can be produced by a set of step-by-step procedures or actions.
• This step-by-step action is called an algorithm.
• The algorithm will process some inputs and produced output.
• Solving problem by computer undergo two phases:

Phase 1:

​ Organizing the problem or pre-programming phase.

Phase 2:

​ Programming phase.

1. Pre-Programming Phase

This phase involves:

  1. Analysing the problem.
  2. Developing the Input-Process-Output (IPO) Chart.
  3. Writing the algorithms.
  4. Drawing the flowchart, or Pseudocode

1.1. Problem Analysis (PAC) Chart

– All these requirements can be presented in a Problem Analysis Chart (PAC)

Data Processing Output
Given in the problem or provided by the user List of processing required or procedures Out requirement

Practicing: Write a problem analysis chart (PAC) that asks a user to enter the distance of a trip in miles, the miles per gallon estimate for the user’s car, and the average cost of a gallon of fuel. Calculate and display the number of gallons of fuel needed and the estimated cost of the trip.

Data Processing Output
distance Calculate fuel fuel needed
miles per gallon Calculate estimated cost estimated cost
cost per gallon

1.2. Input Process Output (IPO) Chart

Has four sections:

​ Input

​ Contains all input data from the problem analysis chart

​ Processing

​ Contains all processing, evident and implied, from problem analysis chart

​ Module/method Reference

​ Contains number for method reference

​ Output

​ All required output as designated by the problem and/or the user

Input Processing Method Reference Output
Distance in miles Enter distance Total fuel needed
Miles per gallon Enter miles per gallon Estimated cost
Cost fuel per gallon fuel needed = distance / miles per gallon
Enter cost fuel per gallon
estimated cost = cost per gallon * fuel needed
Display total fuel and estimated cost
End

1.3. Flowchart

Miles per gallon = the driving distance by consuming a gallon fuel

Already known: Total distance (TD) & Miles per gallon (MG);

Purpose: Figure out the amount of oil required (Needed);

Method: Needed = TD / MG;

1.4. Pseudocode

​ It is used in place of symbols or a flowchart to describe the logic of a program

​ Pseudocode is close to the actual programming language.

2. Programming / Implementation Phase

Transcribing the logical flow of solution steps in flowchart or algorithm to program code and run the program code on a computer using a programming language.

• Programming phase takes 5 stages:

  1. Coding
  2. Compiling
  3. Debugging
  4. Run or Testing
  5. Documentation and maintenance

• Coding
Translation operation in the flowchart or algorithm (pseudocode) into programming language.

​ follow the format of the chosen programming language.

​ Many types or levels of computer programming language such as:

​ Machine language

​ Symbolic language or assembly language

​ Procedure-oriented language

​ The first two are called low-level programming language. The last one is called high-level

programming language.

• Compiling

​ Compiling is a process of a compiler translates a program written in a particular high–level programming language into a form that the computer can execute.

​ The compiler will check the program code also known as source code so that any part of the source code that does not follow the format or any other language requirements will be flagged as syntax error.

• Debugging

​ This syntax error in also called bug, when error is found the programmer will debug or correct the error and then recompile the source code again.

​ The debugging process is continued until there is no more error in the program.

• Run or Testing

​ The program code that contains no more error is called executable program. It is ready to be tested.

​ When it is tested, the data is given and the result is verified so that it should be produced output as intended.

​ Though the program is error free, sometimes it does not be produced the correct result. In this case the program faces logic error.

​ Incorrect sequence of instruction is an example that causes logic error.

• Documentation and Maintenance

​ When the program is thoroughly tested for a substantial period of time and it is consistently producing the right output, it can be documented.

​ Documentation is important for future reference. Other programmer may take over the operation of the program and the best way to understand a program is by studying the documentation.

​ Trying to understand the logic of the program by looking at the source code is not a good approach.

​ Studying the documentation is necessary when the program is subjected to enhancement or modification.

​ Documentation is also necessary for management use as well as audit purposes.
